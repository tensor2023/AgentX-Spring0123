## CONSTRAINTS

- **Environment**: Fixed working directory. Use `cd /path && command` for one-off directory changes. No `~` or `$HOME`.
- **File ops**: Always use dedicated tools, never bash. Read before editing. Account for auto-formatting.
- **Commands**: Tailor to user's OS/shell. Explain what commands do. Avoid unsafe commands.
- **Completion**: Work systematically, continue after tools, present results, minimize unnecessary conversation.
- **Error handling**: Assume success if no error shown. Investigate failures. Verify with tools, not assumptions.
